# 総人口データソース

## 取得元

- 提供元：総務省統計局
- 配信：政府統計の総合窓口 e-Stat（公式Excelファイル）
- 政府統計コード：`00200524`
- 統計：人口推計
- 提供分類：各月1日現在人口
- 統計表：`（参考表）全国人口の推移`
- 公表ページ：<https://www.stat.go.jp/data/jinsui/index.html>
- e-Stat統計表：<https://www.e-stat.go.jp/stat-search/files?cycle=1&data=1&layout=dataset&metadata=1&page=1&sort=year_month+desc&tclass1=000001011678&toukei=00200524&tstat=000000090001>
- 2026年6月公表分の統計表ID：`000040462410`

人口推計は、国勢調査人口を基準人口とし、その後の出生・死亡による自然動態、入国・出国などによる社会動態を用いて、毎月1日現在の人口を算出する基幹統計である。総人口には、日本人人口と、国内滞在期間が3か月を超える外国人人口が含まれる。

## KOKUSEIで表示する値

KOKUSEIは「全国」「総人口」「確定値」のみを表示する。概算値は最新月を示せるが、利用可能な速報資料等を用いた値であり、約5か月後に確定値へ更新されるため、MVPでは採用しない。

Excelの「人」単位の整数を、既存UIの万人単位へ10,000で割る。この変換は浮動小数点を使わず10進文字列として行う。例：`123165360`人 → `12316.536`万人。画面には「出典：人口推計（総務省統計局）、e-Statを基にKOKUSEIが単位変換」と明示する。

## 更新頻度

原則月次。毎月の公表では当月の概算値と、おおむね5か月前の確定値が提供される。当該表はファイル統計で、e-Stat APIの`getStatsData`対象ではないため、公式配布Excelを取得する。既定値は2026年6月公表分で、更新時は`ESTAT_POPULATION_STAT_INF_ID`と`ESTAT_POPULATION_PUBLISHED_AT`を差し替える。import処理は表に含まれる確定値だけを保存する。Webリクエスト時にはe-Statへ接続せず、検証・保存済みの値だけを表示する。

## 検証と障害時の扱い

保存前にXLSX形式、統計表ID、表題、全国・総人口・確定値、人単位、対象年月、公表日、数値形式・妥当範囲、公式HTTPS出典URLを検証する。ファイル全体は保存しない。取得または検証に失敗した場合はDBトランザクションを開始せず、既存値を維持する。

同一対象期間・同一値はno-opとする。同一期間の改定、または新しい期間で直前の公式値から変化した場合だけ、変更前後を`update_histories`へ記録する。公式値と開発用seedは`data_origin`で区別し、公式値が存在する総人口では公式系列だけをAPIへ返す。

## 利用条件

公式Excelの取得にe-Stat APIアプリケーションIDは不要である。`.env`に設定した`ESTAT_APP_ID`は、将来API対象の統計を追加する場合にのみ使用し、Gitへ保存しない。取得は手動実行を前提とし、短時間に繰り返しアクセスしない。

数値データは商用利用を含め利用できる。利用時は出典を表示し、単位変換などの加工を行った場合は加工したことを明記する。
